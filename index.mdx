# Welcome to Inbound

Inbound is a powerful email processing API that allows you to receive, process, and route emails programmatically. Transform any email address into a webhook endpoint and build sophisticated email-driven workflows.

## What is Inbound?

Inbound provides a simple yet powerful way to:

- **Receive emails** at custom email addresses on your verified domains
- **Process emails** through webhooks with full email content and metadata
- **Route emails** based on custom rules and logic
- **Scale email processing** with reliable delivery and retry mechanisms

## Getting Started

<Steps>
  <Step title="Create an Account">
    Sign up for an Inbound account and verify your email address.
  </Step>
  
  <Step title="Verify a Domain">
    Add and verify a domain you own to start receiving emails.
  </Step>
  
  <Step title="Get Your API Key">
    Generate an API key from your dashboard settings.
  </Step>
  
  <Step title="Install the SDK">
    Install the TypeScript SDK or use the REST API directly.
    
    ```bash
    npm install exon-inbound
    ```
  </Step>
  
  <Step title="Create Your First Email">
    Set up an email address and webhook to start processing emails.
    
    ```typescript
    import { createInboundClient } from 'exon-inbound'
    
    const inbound = createInboundClient({
      apiKey: 'your_api_key_here'
    })
    
    // Create a webhook
    const webhook = await inbound.createWebhook({
      name: 'Email Processor',
      endpoint: 'https://api.example.com/webhook'
    })
    
    // Create an email address
    const email = await inbound.createEmail({
      domain: 'example.com',
      email: 'hello@example.com',
      webhookId: webhook.id
    })
    ```
  </Step>
</Steps>
